Audience / Goal

My goal is to make a portfolio that can communicate important information about each of my projects quickly and effectively. I have been hearing from a lot of recruiters that they have sometimes hundreds of applications to look through, so I wanted to make it easy for them to find and parse information. Because of this, I decided to break information into blocks for each of my projects, as well as a simple nav bar. I haven't added all the content and whatnot that I want in my final portfolio, but the style that I want is present so future additions shouldn't be difficult.

Planning

grid layout sketch for part 1 of the project responsiveness sketch for part 2 of the project

To add responsiveness, I plan create a wide design that seperates the many content boxes so they are in a neater layouy at a desktop width. The current layout will be used for a shorter width / mobile width. I would primarily use flexbox, to resize the content as the browser width changes. A breakpoint would change the from the wide flexbox version to the condensed grid version like in the first part of the project.

Execution

Creating and adding content to my site, I realized I would not be able to use the complex mish-mash grid from the spec portion. I decided to make re-usable classes to lay out important information in a customizable way. I found that this was a lot more practical for adding new information in html. The responsiveness I added mostly changed some values, but also changed text alignement for headers and collapsed the project cloumns to display one project per row.
